# L8Learn Amendment: AI-Generated Content — The AI IS the Teacher

## Purpose

The current system treats content as a **static library** of pre-authored activities that the adaptive engine selects from. This is fundamentally wrong. The AI should **generate** lessons, materials, screen activities, and worksheets in real-time — personalized to each student's profile, level, interests, learning style, and attention span.

The AI replaces the teacher. It doesn't pick from a catalog — it creates the lesson.

---

## 1. What Changes

### Before (Static Content Library)
```
Content authors create 1200 Activities → stored in database
  → Adaptive engine picks Activity ACT-0042 for Jake
  → Jake does "Question 1 for this activity?" with options A/B/C/D
  → Same activity for every student at this level
```

### After (AI-Generated Content)
```
Jake's profile: Grade 4, multiplication 7s at DEVELOPING,
  visual learner, 8-min attention, uses tally marks, likes dinosaurs

AI generates "Dinosaur Array Adventure":
  → Lesson plan: physical counting → screen arrays → practice problems
  → Materials: "Get 7 groups of 8 small objects (blocks, coins, toys)"
  → Screen activity: interactive array builder with dinosaur theme
  → 4 practice problems: 7×3, 7×6, 7×8, 7×4 (targeting 7s specifically)
  → Worksheet PDF: 6 problems with dinosaur border, grid paper included
  → Hints: "Try building an array — 7 rows, 8 columns. Count the total!"
  → Duration: 8 minutes (matches attention profile)
  → Success criteria: 3 of 4 correct → advance, <2 → scaffold with easier
```

## 2. New Protobuf: learn-generated.proto

```protobuf
syntax = "proto3";
package learn;
option go_package = "./types/learn";
import "l8common.proto";
import "api.proto";
import "learn-content.proto";

// ============================================================================
// AI-GENERATED LESSONS
// ============================================================================

enum GeneratedLessonStatus {
    GENERATED_LESSON_STATUS_UNSPECIFIED = 0;
    GENERATED_LESSON_STATUS_GENERATING = 1;
    GENERATED_LESSON_STATUS_READY = 2;
    GENERATED_LESSON_STATUS_IN_PROGRESS = 3;
    GENERATED_LESSON_STATUS_COMPLETED = 4;
    GENERATED_LESSON_STATUS_SKIPPED = 5;
}

// @PrimeObject
// A complete lesson generated by the AI for one student at one point in time.
// Not reusable — generated fresh each time based on the student's current state.
message GeneratedLesson {
    string lesson_id = 1;
    string student_id = 2;
    string path_id = 3;                      // Which LearningPath this serves
    GeneratedLessonStatus status = 4;

    // What this lesson teaches
    repeated string skill_ids = 5;           // Target skills
    SubjectType subject = 6;
    DifficultyLevel difficulty = 7;
    string topic = 8;                        // "Multiplication by 7s"
    string theme = 9;                        // "Dinosaurs" (from student interests)

    // Lesson structure (AI-generated)
    string title = 10;                       // "Dinosaur Array Adventure"
    string objective = 11;                   // "Student will multiply by 7 using arrays"
    repeated LessonStep steps = 12;          // Ordered sequence of activities
    repeated string materials_needed = 13;   // Physical materials for hands-on steps
    int32 estimated_minutes = 14;            // Based on student's attention profile
    string parent_instructions = 15;         // For homeschool: what the parent should do

    // Generated worksheet (optional)
    string worksheet_content = 16;           // JSON: problems, layout, instructions
    string worksheet_pdf_path = 17;          // Generated PDF via Layer8FileUpload

    // Success criteria
    int32 min_correct_to_advance = 18;
    int32 min_correct_to_pass = 19;
    string on_struggle_strategy = 20;        // "scaffold", "alternate_activity_type", "break"

    // Results (filled after student completes)
    int32 questions_correct = 21;
    int32 questions_total = 22;
    int32 actual_minutes = 23;
    string ai_observation = 24;              // AI analysis of how the student performed

    // Metadata
    int64 generated_at = 25;
    int64 started_at = 26;
    int64 completed_at = 27;
    map<string, string> custom_fields = 28;
    l8common.AuditInfo audit_info = 29;
}

message GeneratedLessonList {
    repeated GeneratedLesson list = 1;
    l8api.L8MetaData metadata = 2;
}

// Embedded child of GeneratedLesson
message LessonStep {
    int32 step_number = 1;
    string step_type = 2;                    // "physical", "screen", "discussion", "worksheet", "break"
    string title = 3;                        // "Build Arrays with Blocks"
    string instructions = 4;                 // Full instructions for this step
    int32 duration_minutes = 5;
    string parent_role = 6;                  // "guide", "observe", "none" (independent)

    // For screen steps: generated questions
    repeated GeneratedQuestion questions = 7;

    // For physical steps: what to do with materials
    string materials_instructions = 8;       // "Arrange 7 groups of 8 blocks..."
}

// Embedded child of LessonStep
message GeneratedQuestion {
    string question_id = 1;
    string prompt = 2;                       // "What is 7 × 8?"
    string prompt_image_desc = 3;            // AI describes an image to generate: "7×8 array of dinosaurs"
    QuestionType question_type = 4;
    repeated GeneratedOption options = 5;     // For choice questions
    string correct_answer = 6;               // For numeric/text questions
    string explanation = 7;                  // "7 groups of 8 = 56. You can count the array!"
    repeated string hints = 8;               // Progressive hints
    DifficultyLevel difficulty = 9;
}

// Embedded child of GeneratedQuestion
message GeneratedOption {
    string option_id = 1;
    string text = 2;
    bool is_correct = 3;
    string feedback = 4;                     // Why this is right/wrong
}
```

## 4. How It Works End-to-End

```
Student logs in
    │
    ▼
Student player checks LearningPath.currentLessonId
    │
    ├── If no current lesson → request new lesson from AI
    │     │
    │     ▼
    │   LLM generates complete lesson based on:
    │     - StudentProfile (learning style, attention, interests, accommodations)
    │     - Current SkillMastery (what to teach next)
    │     - Skill prerequisite graph (what's ready to learn)
    │     - AdaptationRules (if any triggered)
    │     │
    │     ▼
    │   GeneratedLesson stored in database
    │   LearningPath.currentLessonId updated
    │   PromptLog: GENERATE_LESSON logged with PII masked
    │
    ▼
Student player renders the lesson:
    │
    Step 1: "Get 7 groups of 8 blocks" (physical, parent guides)
    │   Parent marks complete → next step
    │
    Step 2: "Build an array on screen" (screen, interactive)
    │   Student drags blocks into array → scored
    │
    Step 3: "Practice problems" (screen, 4 questions)
    │   Each answer recorded as Interaction
    │   Hints available per student's hint_style preference
    │
    Step 4: "Print worksheet" (optional, parent prints)
    │   Worksheet generated from lesson's worksheetContent
    │   Student fills in on paper → scan back later
    │
    ▼
Lesson complete:
    - Score calculated (questions_correct / questions_total)
    - If >= minCorrectToAdvance → AI generates next lesson (harder)
    - If >= minCorrectToPass → repeat at same level with different content
    - If < minCorrectToPass → AI generates scaffold lesson (easier, different approach)
    - SkillMastery updated
    - StudentProfile updated (OnSessionComplete)
    - Results feed into next lesson generation
```

## 5. What This Replaces

The static Content module (Course → Unit → Lesson → Activity) becomes **optional** — a curriculum framework for administrators to define scope and sequence. The actual content the student sees is generated by the AI.

| Before | After |
|--------|-------|
| 20 pre-authored Courses | Curriculum standards (scope & sequence) — defines WHAT to teach |
| 100 pre-authored Units | Skill graph — defines prerequisite ORDER |
| 400 pre-authored Lessons | AI generates lessons on demand — defines HOW to teach THIS student |
| 1200 pre-authored Activities | AI generates activities within each lesson — personalized questions, themed to interests |
| Static worksheets | AI generates worksheets per lesson — at student's exact level |
| Same content for every student | Every student gets unique content matched to their profile |

The Skill graph remains important — it tells the AI WHAT to teach next. But the AI decides HOW to teach it, at what difficulty, with what theme, using what materials, for how long.

### ServiceCallback

- **After POST** (lesson generated): Update LearningPath.currentLessonId. Log GENERATE_LESSON to PromptLog.
- **After PUT** (lesson completed): Score results. Update SkillMastery. Trigger ProfileUpdater.OnSessionComplete. If minCorrectToAdvance met → generate next lesson (advance). If not → generate scaffold lesson. Update LearningPath.

## 7. Student Player Changes

The student player stops loading from the static Activity database. Instead:

1. On load: fetch `GeneratedLesson` for current path
2. If none exists or current lesson completed: POST to generate a new one
3. Render steps sequentially (physical → screen → worksheet)
4. Screen steps render `GeneratedQuestion` (same renderer as before)
5. Physical steps show instructions + "Mark Complete" button
6. On lesson complete: PUT results back

The existing `player-renderer.js` works for screen questions — `GeneratedQuestion` uses the same `QuestionType` enum as the old `Question` type.

## 16. Immutability

GeneratedLesson is **NOT immutable** — it has a lifecycle:
1. GENERATING → created by AI (POST)
2. READY → AI finished generating (PUT by engine)
3. IN_PROGRESS → student started (PUT by player)
4. COMPLETED → student finished, results saved (PUT by player)

PUT is allowed (to record results), DELETE is not (audit trail). The `onStruggleStrategy`, `questions`, and `steps` fields are immutable after generation — only `status`, `questionsCorrect`, `questionsTotal`, `actualMinutes`, `aiObservation`, `startedAt`, `completedAt` change on PUT.

### 18.2 Content Safety Filter

New file: `go/learn/adaptive/engine/content_safety.go` (~100 lines)

The LLM generates content shown directly to children. Before marking a lesson READY, scan for:
- Profanity / inappropriate language (word list)
- Violence, weapons, drugs references
- Bias or stereotyping in problem context
- Mathematical errors (verify answers are correct for generated problems)
- Prompt injection attempts in generated text

```go
type ContentSafety struct {
    blockedWords []string
}

func (cs *ContentSafety) ValidateLesson(lesson *learn.GeneratedLesson) []string {
    var issues []string
    for _, step := range lesson.Steps {
        issues = append(issues, cs.scanText(step.Instructions)...)
        for _, q := range step.Questions {
            issues = append(issues, cs.scanText(q.Prompt)...)
            issues = append(issues, cs.scanText(q.Explanation)...)
        }
    }
    return issues // empty = safe
}
```

**Flow**: AI generates lesson → `ContentSafety.ValidateLesson()` → if issues found, lesson marked GENERATING (retry with modified prompt) → if clean, marked READY.

### 18.3 Pre-Generation — Never Let the Student Wait

Follow l8alarms escalation scheduler pattern: pre-compute future items when current item is created.

**When student starts a lesson** (status → IN_PROGRESS):
- Trigger background generation of the NEXT lesson
- By the time student finishes current lesson, next is already READY

**When student completes a lesson** (status → COMPLETED):
- If next lesson is READY → assign immediately (no wait)
- If next lesson is still GENERATING → show brief "preparing..." (rare)
- If next lesson failed → generate on demand (fallback)

**Pre-generation buffer**: Always keep 2 lessons ahead. When student has 1 completed + 1 in_progress + 0 ready → trigger generation. This ensures the student NEVER waits.

### 18.4 LLM Failure Fallback

When the LLM fails (timeout, rate limit, error response):

1. **Retry once** with simplified prompt (fewer context fields)
2. **If retry fails** → generate a **template lesson** from the static content library:
   - Find Activities matching the target skill + difficulty in the existing database
   - Wrap them in a GeneratedLesson shell with generic instructions
   - Mark as `theme: "fallback"` so the system knows it wasn't AI-generated
3. **If no static content matches** → show a practice review of previously mastered skills (always available)

```
LLM Call → success → lesson READY
  ↓ fail
Retry → success → lesson READY
  ↓ fail
Static content fallback → lesson READY (theme: "fallback")
  ↓ no match
Review of mastered skills → lesson READY (theme: "review")
```

The pre-generation buffer (18.3) makes this rare — the fallback only triggers if the LLM was down when the pre-generation ran AND the student finishes the current lesson before it recovers.

### 18.5 Cost Tracking and Budget Enforcement

**PromptLog already tracks tokens.** Add cost estimation:

Add to `learn-llm.proto` LLMConfig:
```protobuf
double cost_per_input_token = 13;       // e.g., 0.000003 for Sonnet
double cost_per_output_token = 14;      // e.g., 0.000015 for Sonnet
double cost_today = 15;                 // Running total
double max_daily_cost = 16;             // Budget limit (e.g., $15.00)
```

Add to `prompt_logger.go`:
```go
estimatedCost := float64(inputTokens)*config.CostPerInputToken +
                 float64(outputTokens)*config.CostPerOutputToken
config.CostToday += estimatedCost
if config.CostToday >= config.MaxDailyCost {
    // Switch mode to LOG_ONLY until tomorrow
    // Existing pre-generated lessons serve students
    // New generation paused until budget resets
}
```

**What happens when budget is hit mid-day:**
- Pre-generated buffer (18.3) keeps serving students
- No new lessons generated until midnight reset
- AI Monitor shows cost warning
- Static content fallback (18.4) used for any student without a buffer

**Cost projection:** At Sonnet pricing ($3/M input, $15/M output):
- ~3000 input + ~1500 output per lesson = ~$0.03/lesson
- 1000 students × 3 lessons/day = 3000 lessons = ~$90/day
- With pre-generation buffer: students share similar skill-level lessons can reuse themes → potential 30-50% reduction

## 20. Compliance

### Data Safety
- Generated lessons contain NO student PII — only skill targets, difficulty, and theme
- Student profile is masked before sending to LLM (same PII pipeline as all other prompts)
- Generated content is logged to PromptLog for review

### COPPA
- AI-generated content uses interests (dinosaurs, space) but never identifies the child
- Guardian can review generated lessons in parent portal
- Content themes respect guardian preferences (AI tutor settings: should_avoid list)

### Duplication Prevention
- `BuildGenerateLessonPrompt` uses the shared `PromptBuilder` — no template copy-paste
- `GeneratedQuestion` reuses the same `QuestionType` enum as the original `Question` — renderer compatibility
- Student player reuses existing `player-renderer.js` for screen questions
